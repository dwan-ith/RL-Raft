How Raft Prevents Unnecessary Elections

To prevent multiple follower nodes from constantly timing out, splitting votes, and triggering inefficient elections, RL-Raft implements a 4-stage pipeline funnel:

1
Followers

All nodes in the cluster start as passive Followers listening for periodic Leader heartbeats (AppendEntries RPCs).

2
Pre-Candidates

When a Follower's randomized timeout expires, it does NOT increment its term immediately. It first becomes a Pre-Candidate to check if the network is healthy and a Leader is truly offline.

3
Candidates

Peer nodes deny pre-votes if they can still hear a Leader or if the Pre-Candidate has stale logs. Only if a Pre-Candidate gathers a majority of pre-votes does it officially advance to the Candidate state and increment the term.

4
Leader

The Candidate requests official votes. If it successfully secures a majority vote, it is crowned the Leader and starts sending heartbeats, transitioning everyone else back to Followers.
